CLI COMMANDS & FLAGS
Session Management
claude Start a new interactive session
claude "query" Start session with an opening prompt
claude -c Continue the most recent session in this dir
claude -r "name" Resume a named session (e.g. "outreach-py")
claude -n "name" Name a new session for easy resume later
claude --resume Interactive picker to choose a past session
claude --fork-session Resume but create a new session ID (safe branch)
Scripting & Automation
claude -p "query" Print mode: run query, output result, then exit
cat file | claude -p "query" Pipe file content into Claude (great for logs)
claude -c -p "query" Continue last session in print/headless mode
--output-format json Return structured JSON output (for pipelines)
--max-turns 3 Limit agentic turns (prevents runaway loops)
--max-budget-usd 2.00 Cap API spend per run (print mode only)
--bare -p "query" Minimal fast mode — skips MCP, plugins, memory
Python Project Workflows
claude --add-dir ../lib Add extra dirs Claude can read (e.g. shared utils)
claude --tools "Bash,Edit,Read" Restrict tools to safe set for code editing
claude --verbose Show full turn-by-turn output for debugging
claude --debug "api" Debug API calls only (filter by category)
claude -w feature-name Isolated git worktree session (parallel work)
claude --effort high Increase reasoning effort for complex logic
claude --model claude-sonnet-4-6 Set model explicitly for session
System Prompt Control
--append-system-prompt "text" Add rules on top of defaults (safest option)
--append-system-prompt-file ./rules.txt Load custom rules from file (e.g. style guide)
--system-prompt "text" Replace entire system prompt (full control)
--system-prompt-file ./prompt.txt Load replacement prompt from file
--settings ./settings.json Load custom settings from a JSON file
Permissions & Safety
--permission-mode plan Plan-only mode, no writes — safe review pass
--allowedTools "Bash(git log *)" Pre-approve specific tool patterns (no prompt)
--disallowedTools "Edit" Block specific tools from being used at all
--dangerously-skip-permissions Skip all prompts (automation only — use carefully)
--no-session-persistence Run without saving session to disk
MCP & Auth
claude mcp Configure MCP servers (n8n, custom tools)
--mcp-config ./mcp.json Load MCP servers from a JSON config file
--strict-mcp-config Only use MCP servers from --mcp-config file
claude auth login Sign in to Anthropic account
claude auth login --console Login via Console (API key billing)
claude auth status Check auth status (exits 0=logged in, 1=not)
claude update Update Claude Code to the latest version
Outreach Scripts cd JobOpsOutreach && claude -n "outreach-session"
Resume a Named Session claude -r "session-name" "pick up where we left off"
Safe Code Review claude --permission-mode plan "review outreach.py for bugs"
Pipe a Log File cat error.log | claude -p "what failed and why"
